1. Auftrag vom Auftraggeber

3. Vorentwicklung
4. Trends
5. Marktanalyse
## Die Planungsphase [Programmieren Lernen - Lerne alle Sprachen in einer Serie]
In der Planungphase wird das Backlog bzw. das Lastenheft und ein Projektplan erstellt, der u. a. auch eine Kostenrechnung beinhaltet.  

Wichtig sind hier auch die Requirements die bspw. wie folgt formuliert werden.  
* Das System soll auf einen computer mit mindestens 1 GB RAM und einer CPU einem Kern und 1 GHz laufen.
* Die druchschnittliche Antwortzeit soll höchsten 10 ms betragen
* Das System berechnet aus den Eingabedaten die Summe.

Generell gibt es folgende Regelungen zur Formulierung von Requirements.
* Korrekt
* Vollständig
* Keine Widersprüche
* In "normaler" Sprache
* Eindeutig
* Validieren! mit Auftraggeber

Um Infos zur Planung zu erhalten, kann man folgende Dinge tun.
* Befragungen
* Interviews
* Aufgabenanalyse
* Szenarios
* Use Cases

Gruppen und Infoquellen zur Ermittlung von Informationen sind folgende.
* benutzer
* Käufer
* Auftraggeber/Sponsoren
* (Politik,
* Soziale Themen,
* ...)
## Requirements Kategorien [Programmieren Lernen - Lerne alle Sprachen in einer Serie]
Requirements lassen sich wie folgt Kategorisieren.
* Funktionale Anforderungen
  * Systemverhalten
  * Daten
  * Input/Output
* Qualitative Anforderungen
  * Performance
  * Zuverlässigkeit
  * Sicherheit
  * Verfügbarkeit
  * Wartbarkeit
* Constraints
  * Legales
  * Kultur
  * Leistung
## Der Entwurf [Programmieren Lernen - Lerne alle Sprachen in einer Serie]
Um Entwürfe zu erstellen kann man folgende Dinge nutzen.
* Systemmodell
* Erstmal nur WAS
  * Lastenheft verfeinern mit Use-Case-Diagrammen
* Später dann WIE
  * Architektur- "modular design"
  * Genaue Strukturen -"object-oriented design"

Es gibt folgende Modellarten um Entwurfe erstellen zu können.
* Funktionsmodell (Lastenheft, Anforderungen)
* Object Model - Vererbungen, Datenbeziehungen
  * UML-Klassendiagramm
* Dynamic Model - Zusammenhänge untereinander
  * Sequenzdiagramm
  * Zustandsdiagramm
  * Aktivitätsdiagramm
